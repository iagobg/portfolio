## Visão geral

O motor de recomendação de laptops é uma vertical slice para ajudar usuários a transformar requisitos em linguagem natural em sugestões fundamentadas. A versão atual é intencionalmente limitada a laptops.

<Decision title="Recomendações fundamentadas">A LLM interpreta intenção, mas a recomendação final precisa vir de uma base real de produtos.</Decision>

## O problema

Uma pessoa pode pedir um laptop para programação, faculdade, jogos, viagens, bateria ou orçamento rígido. Esses pedidos são ambíguos, mas a recomendação precisa retornar produtos reais com motivos claros.

## Arquitetura

O sistema separa interpretação de ranking. O modelo ajuda a extrair orçamento, uso e preferências. A camada de ranking aplica filtros e regras fundamentadas contra a base de laptops.

| Etapa | Responsabilidade |
| --- | --- |
| Interpretação | Converter linguagem em restrições estruturadas |
| Recuperação | Selecionar laptops candidatos em registros reais |
| Ranking | Equilibrar orçamento, desempenho, bateria e portabilidade |
| Explicação | Descrever por que cada resultado se encaixa |

## Decisões técnicas

Manter a LLM longe da invenção final de produtos é a decisão central. Ela pode explicar, classificar e extrair, mas a lista de produtos continua vinculada ao banco.

<Callout title="Escopo vertical">A implementação atual foca apenas em laptops. A arquitetura deixa espaço para categorias futuras, mas o comportamento público deve ser honesto sobre o escopo atual.</Callout>

## Aprendizados

Sistemas de recomendação precisam de disciplina de produto tanto quanto de capacidade de modelo. Separar interpretação por modelo do ranking determinístico torna a aplicação mais testável, explicável e segura.
